1- Javascript a differenza dei linguaggi CSS e HTML, è un linguaggio di programmazione, più specificamente un linguaggio di scripting. Grazie a Javascript possiamo quindi rendere interattive le nostre pagine web, attraverso eventi, funzioni ecc. Javascript a differenza di molti altri linguaggi di programmazione non ha bisogno di avere un compilatore per essere interpretato, ma esso è infatti interpretato direttamente dal browser web. Javascript è un linguaggio di programmazione molto diffuso ed è presente nella maggior parte delle pagine web. Esso è definito un linguaggio di programmazione con una tipizzazione di "basso livello", poichè non è obbligatorio definire il tipo di dato ad una variabile ad esempio.
Con Javascript possiamo elaborare dati, eseguire operazioni, comunicare server esterni, creare funzioni, eventi ecc. Esso quindi ci permetti di manipolare il DOM(Document Object Model), ossia la pagina HTML, nella quale creiamo appunto eventi, come ad esempio rendere interattivo un bottone. Come tutti i linguaggi di programmazione anche in Javascript sono presenti le variabili e le costanti. La particolarità di esse sta nel fattto che possiamo dichiarare le variabili in due modi: Let e Var; Invece le costanti con const. Let è stata introdotta da poco in Javascript, essa si distingue da var sostanzialmente per una questione di Scope. La variabile Let infatti è disponibile solo all'interno del blocco di codice in cui è stata dichiarata. Invece la variabile var è disponibile all'interno dell'intera funzione in cui sono state dichiarate. Invece la costante const, si differienza dalle variabili, poiché appunto una volta assegnata non può essere modificata.

2- CSS sta per (Cascade Style Sheet) è un linguaggio di stile, che serve per dare appunto uno stile alla pagina web. Come l'HTML anche questo linguaggio NON è un linguaggio di programmazione. Il CSS tramite attributi ci permette di modificare i componenti importati dalla pagina HTML. Ad esempio se importiamo un tag paragrafo dall'HTML, tramite CSS è possibile assegnargli un colore in questo modo: p { color:red }; In questo esempio abbiamo fornito il colore rosso al testo del paragrafo. Per ordine all'inizio di un file CSS, è opportuno assegnargli delle istruzioni di reset. Per puntare all'intera pagina si usa il simbolo *. Quindi un esempio di CSS reset è il seguente: *{ padding: 0; margin: 0; box-sizing: border-box;}. Tramite queste istruzioni abbiamo dato quindi il valore 0 al padding, il valore 0 ai margini, e nell'ultima istruzione specifichiamo che tutti i vari elementi che compongono la pagina rientrino nello spazio del layout, portando con se anche altre proprietà tipo la larghezza, il padding e i bordi. Inoltre il CSS è usato per dimensionare la pagina web a seconda dello schermo che un utente utilizza, quindi rendere "responsive" le pagine web, utilizzando le media query.

3- HTML sta per (Hypertetx Markup Language) è un linguaggio di markup, che serve per creare le pagine web. In sostanza HTML crea lo "scheletro" della pagina web, dopodiché è possibile collegare file Javascript e file CSS per aggiungere script o gestire lo stile del layout della pagina. HTML non è un linguaggio di programmazione quindi, poiché non contiene logica e non è possibile eseguire operazioni.
Una pagina HTML è formata da TAG per funzionare. Un tag è un elemento che serve per creare un qualsiasi componente, ad esempio il tag 'p' serve per creare un paragrafo di testo, un tag 'h1' serve per creare un titolo e così via. In HTML esistono i tag semantici, ossia quei tag che servono per ordinare al meglio il progetto.
Ad esempio quando troviamo il tag "header" capiamo subito che all'interno di questo tag ci saranno tutti gli elementi per creare la parte alta del sito, ad esempio una navbar. E' opportuno dare una classe o un id ad un tag in HTML, per poi modificarlo in CSS. Per classe intendiamo un attributo, che ci serve per poi modificare tutti i tag con la stessa classe in CSS. Per id intendiamo sempre un attributo, ma a differenza delle classi, l'id è univoco al singolo tag assegnato, per poi andarlo a modificare in CSS. Inoltre troviamo nella parte alta del progetto il tag "head", da non confondere con "header", nel quale sono presenti tutti i metadati che consentono al file html di comunicare con il browser web. Quindi in sostanza tutte le pagine web sono formate dal linguaggio HTML, cioè lo scheletro della pagina, al quale possiamo collegare i file CSS per dare lo stile alla pagina, e i file Javascript per dare appunto funzionalità aggiuntive alla pagina. Infine è bene ricordare che l'HTML NON è un linguaggio di programmazione.

4- Angular sono presenti i componenti. Per componente intendiamo costrutti modulari che combinano logica e UI (User Iterface) in un singolo elemento. In altre parole sono delle parti dell'interfaccia utente che possono essere riutilizzate all'interno dell'applicazione. Ogni componente ha una classe typescript che definisce la logica e un template HTML che definisce la struttura. I componenti in angular inoltre seguono il pattern architetturale noto come component-based architecture, che consente una maggiore scalabilità e una maggiore facilità di manutenzione. Attraverso il routing è possibile collegare i singoli componenti tramite il decorator @component. Il routing consente quindi di navigare tra le diverse pagine del progetto. Si basa su un concetto di route, ossia un collegamento tra un url e una vista dell'applicazione. Ogni route viene definita all'interno di un modulo. Quando si va a digitare l'Url corrispondete alla route, Angular carica la vista associata alla route e la visualizza l'utente. Quindi in Angular possiamo definire il routing come un meccanismo fondamentale per la navigazione tra le diverse parti dell'applicazione.

5- La differenza sostanziale tra i linguaggi tipizzati e non è per la gestione dei tipi di dati.
Javascript come spiegato in precedenza è un linguaggio di programmazione non tipizzato, il che significa che le variabili non hanno un tipo di dato definito, quindi puoi assegnare un valore di qualsiasi tipo a una variabile senza dichiarare esplicitamente il tipo di dato.
Ad esempio una variabile può contenere un numero inizialmente e una stringa in un secondo momento senza che il programma riporti errori.
Typescript invece è un linguaggio di programmazione tipizzato, definito come un "superSet" di Javascript, aggiungendo quindi il concetto di "forte tipizzazione". In typescript è necessario dichiarare i tipi di dati alle variabili. Questa dichiarazione iniziale dei tipi consente di avere una maggiore organizzazione del codice. E' necessario ricordare che il codice Javascript valido è anche codice Typescript validio; ad esempio è possibile utilizzare librerie di JS in un progetto TS senza doverli riscrivere
